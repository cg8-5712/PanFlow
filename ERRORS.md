# PanFlow API 错误码定义

## 响应格式

所有接口统一返回格式：

```json
{
  "code": 200,
  "message": "请求成功",
  "data": {}
}
```

**说明：**

- `HTTP 状态码`：表示请求层面的成功或失败
- `code`：业务错误码
- `message`：可读提示信息

---

## 1. 成功

| Code | HTTP | 含义 |
|------|------|------|
| 200 | 200 | 请求成功 |

## 9. Go 代码使用示例

### 响应结构定义

文件：`internal/handler/response.go`

```go
type Response struct {
    Code    int         `json:"code"`
    Message string      `json:"message"`
    Data    interface{} `json:"data"`
}

func Success(c *gin.Context, data interface{}) {
    c.JSON(200, Response{
        Code:    200,
        Message: "请求成功",
        Data:    data,
    })
}

func Fail(c *gin.Context, httpStatus int, code int, msg string) {
    c.JSON(httpStatus, Response{
        Code:    code,
        Message: msg,
        Data:    nil,
    })
}
```

### 使用示例

```go
// 成功响应
response.Success(c, gin.H{
    "token": "xxx",
    "expires_at": "2025-01-01 00:00:00",
})

// 错误响应
if err != nil {
    response.Fail(c, 500, 20011, "获取分享链接信息失败")
    return
}

// 参数错误
if surl == "" {
    response.Fail(c, 400, 20005, "参数错误：surl 不能为空")
    return
}

// 黑名单拦截
if isBlocked {
    response.Fail(c, 400, 20014, "处于黑名单中")
    return
}
```
